## Example: good H2 section (outcome-first)

**Do this:**

> ## File permission change auditing
>
> When a user or admin modifies file permissions, [Product] records the change in real time — capturing who made the change, which object was affected, the old and new permission values, and the workstation it originated from. You can generate an on-demand report or configure an alert to notify your security team immediately.
>
> - View a full audit trail of all permission changes across your file servers
> - Filter by user, server, or time range to isolate suspicious activity
> - Trigger automated alerts when sensitive directories are modified

**Not this:**

> ## File permission change auditing
>
> The Permission Change Report contains the following fields: user name, target object, old permissions, new permissions, workstation name, time of change.
